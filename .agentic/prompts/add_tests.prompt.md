# Prompt: Add or Improve Tests (Enterprise Grade)

## Purpose
Increase test coverage and improve test quality.

## Preconditions
- Code exists without adequate tests.
- Test Coverage Skill is available.

## Required Inputs
- Module or feature to test
- Known gaps (optional)

## Steps
1. Analyze test coverage using Test Coverage Skill.
2. Identify missing:
   - Unit tests
   - Integration tests
   - Edge cases
   - Error paths
3. Generate required tests.
4. Validate tests against testing standards.
5. Summarize improvements.

## Acceptance Criteria
- Coverage meaningfully increases.
- Tests are deterministic and isolated.
- Tests follow naming and structure conventions.

## Output Format
A markdown summary including:
- Coverage gaps found
- Tests added
- Remaining risks
