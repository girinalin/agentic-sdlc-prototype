# Prompt: Implement API Endpoint (Enterprise Grade)

## Purpose
Implement a new API endpoint following architecture, testing, and SDLC standards.

## Preconditions
- A feature plan exists.
- Endpoint requirements are clear.

## Required Inputs
- Endpoint name
- Request/response schema
- Business logic requirements

## Steps
1. Review the feature plan.
2. Generate or update:
   - Pydantic models
   - Service layer logic
   - API route
3. Add unit tests for service logic.
4. Add integration tests for API behavior.
5. Validate code against coding standards.
6. Summarize changes.

## Acceptance Criteria
- Endpoint compiles and runs.
- Unit tests cover success and failure paths.
- Integration tests validate full request/response flow.
- No unrelated files are modified.

## Output Format
A structured summary including:
- Files created/updated
- Code snippets
- Test cases added
- Validation notes
